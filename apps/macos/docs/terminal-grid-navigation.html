Notes for this exploration
New direction:
Rigid grid that maximizes space inside one "context" / workspace. No overflow, no complex scrolling within a workspace. When you outgrow the comfortable grid, create another workspace/context instead of one giant canvas.
Benefits:
No overlapping, viewport always in context, simple maximize. Switch contexts (tabs/workspaces) to see more terminals.
This prototype mirrors the current Swift implementation (virtualW/H based on target cell × effCols/effRows, max(available, virtual) logic, gutters, HudTiling row-major placement). The goal is to quickly explore what "excellent scrolling + awareness" should feel like before more SwiftUI work.
