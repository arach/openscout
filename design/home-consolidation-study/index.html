Scout — Home Consolidation Study

Static explorations for a denser, always-on fleet homepage. Grounded in packages/web/client/screens/home (content.tsx, HomeHero.tsx, fleet-home.css quiet panels). No production code changed.

Problem. Local tail + Message composer only appear in the quiet/idle state (QuietStartPanel). When the fleet is active they vanish, and repo/PR/study/port context lives elsewhere. This study keeps the operator-console language and asks how much “desk surface” home can hold before it overcrowds.
Option A · Minimal pole

Quiet Desk

Hero + two always-on panels (Message · Local tail). Moving and activity collapse to a thin status strip. Judging floor: “is two enough?”

 Option B · Mid

Console Strip

Hero, always-on Message/Tail row, then a compact context strip (repo · PRs · studies · ports) above What’s moving + live activity.

 Option C · Dense pole

Mission Grid

Hero + 2×3 panel grid (Message, Tail, Repo, PRs, Studies, Ports) with moving + stream below. Overcrowding-line test.

 Option D · Split

Split Command

Two-column ops desk: left work surface (compose · tail · stream), right situational stack (moving · repo · PRs · studies · ports).
